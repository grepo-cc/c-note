- ### 计算机系统的组成包括<u>硬件系统</u>和<u>软件系统</u>
 
=== "计算机部件"

	=== "冯·诺依曼体系结构"
		<!-- ![](net@/CPU组成.svg ) -->
		???+ pied-piper inline  "冯·诺依曼体系结构"		
			- 计算机硬件由运算器()、控制器（CU）、存储器(Memory)、输入设备(Input)和输出设备(Output),五大部分组成	
			- 中央处理器CPU包含运算器和控制器	
			- 计算机运行时，运算器的操作和操作种类由控制器决定。运算器处理的数据来自存储器；处理后的结果数据通常送回存储器，或暂时寄存在运算器中
		 

	=== "运算器组成" 
		???+ abstract "运算器组成"
			- 组成：算术逻辑单元（Arithmetic and Logic Unit）、累加器、状态寄存器、通用寄存器组等
			- 算术逻辑运算单元(ALU)的基本功能为加、减、乘、除四则运算，与、或、非、异或等逻辑操作，以及移位、求补等操作				
			- 状态寄存器：用来标识协处理器中指令执行情况的，它相当于CPU 中的标志位寄存器 又名条件码寄存器（SR,Status register）
			??? tip "状态寄存器用来存放两类信息"
				- 一类是体现当前指令执行结果的各种状态信息（条件码），如有无进位（CF位）、有无溢出（OF位）、结果正负（SF位）、结果是否为零（ZF位）、奇偶标志位（P位）等；
				- 另一类是存放控制信息（PSW:程序状态字寄存器），如允许中断(IF位)、跟踪标志（TF位）等。有些机器中将PSW称为标志寄存器FR（Flag Register）

			- 累加寄存器AC：主要用来保存操作数和运算结果等信息，从而节省读取操作数所需占用总线和访问存储器的时间
			- 数据缓冲寄存器DR：CPU从存储器中获取的数据优先存在数据缓冲寄存器中	
			
	=== "控制器"
		???+ abstract "控制器"
			- 组成：由指令寄存器IR(InstructionRegister)、程序计数器 PC(ProgramCounter)和操作控制器OC(OperationController)等  
			- 计数器用途  
			- 程序计数器PC：用于存放下一条指令所在单元的地址。ARM型CPU中为程序计数器PC，X86型CPU中为指令指针寄存器IP，用于控制程序中指令的执行顺序
			- 指令寄存器IR：存放正在执行的指令
			- 地址寄存器：可作为存储器指针
		
	=== "存储器"
		![](:/6d7af8c2ca774d5399b974b88c2bace1)
		???+ abstract "存储器"
			[见存储器????](#存储器)
			
			- 存储器：内存存储机制		
			
			
			
			- 根据存储器在计算机系统中所起的作用，可分为主存储器、辅助存储器、高速缓存存储器、控制存储器等
			- 为了解决对存储器要求容量大，速度快，成本低 三者之间的矛盾，目前通常采用多级存储器体系结构，即使用高速缓冲存储器、主存储器和外存储器
			- 高速缓冲存储器:Cache 高速存取指令和数据 存取速度快，但存储容量小
			- 主存储器:内存 存放计算机运行期间的大量程序和数据 存取速度较快，存储容量不大
			- 外存储器: 外存 存放系统程序和大型数据文件及数据库 存储容量大，成本低
			- 高速缓存的意义：为了解决高速设备和低速设备相连，提高访问速度
=== "指令系统"
	=== "指令"
		???+ abstract "指令"
			- 指令集:一台计算机中所有机器指令的集合
			- 指令周期是执行一条指令所需要的时间，一般由若干个机器周期组成，是从取指令、分析指令执行完所需的全部时间。
			- 指令的定义:一条指令可分解为操作码和地址码两部分，操作码确定指令的操作类型，地址码确指令所要处理操作数的位置。
			- 指令字长
			- 指令分类
			- 指令存储
			- 指令格式
	=== "指令和操作数的寻址方式："
		???+ abstract "指令和操作数的寻址方式："
			- 立即寻址(立即数寻址)：直接指出操作数本身，直接访问内存，时间最短； 
			- 变址寻址
			- 直接寻址(直接内存寻址)
			- 间接寻址  
			- 寄存器寻址
			- 寄存器间接寻址
	=== "流水线"
		???+ abstract "流水线"
			- 流水线技术：取址—分析—执行，流水线是指在程序执行时多条指令重叠进行操作的一种准并行处理实现技术。即可以同时为多条指令的不同部分进行工作，以提高各部件的利用率和指令的平均执行速度	

				!!! example	annotate ""
					$\sum_{i = 1}^{n}Δt_i +(M -1)Δt_{max}$  (1)				 		
				1.  流水线由n组成，完成M个任务的实际时间  
					Δt^i^流水线周期各个时间段  
					Δt~max~:流水线周期最长的那一段的执行时间
										
			- 流水线技术指标:
				- T~k~流水线方式时间  
				- T~s~不使用流水线所用的时间   
				- 效率:流水线的各个部件的利用率
				!!! example	annotate "吞吐率"
					$( T_p) = n/T_k$  (1)				 		
				1.  吞吐率:指的是计算机中的流水线在单位时间内可以处理的任务或执行指令的个数，理论上的大吞吐率是：1/流水线周期   
					n:指令条数
							
				!!! example	annotate "加速比"
					$ (S) = T_s/T_k$  (1)				 		
				1.  加速比:完成一批任务，串行模式所用的时间与使用流水线所用的时间之比

##### 硬件系统分为三种典型结构

>- 单总线结构
>- 双总线结构
>- 采用通道的大型系统结构
  
##### 总线：包括数据总线、地址总线、控制总线 
 
##### 原码，补码，反码
>- 计算机储存有符号的整数时，是用该整数的补码进行储存的
>- 0的原码、补码 都是0
>- 正数的原码、补码可以特殊理解为相同
>- 负数的补码是它的反码加1
